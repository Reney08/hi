Anleitung und API-Spezifikation: BarBot Steuerung

Diese Dokumentation beschreibt die Schnittstellen zwischen Web-App, Hauptserver (Flask) und der ESP-Hardware.

🛠️ Hardware-Spezifikation fuer den ESP

Der ESP muss als HTTP-Server auf Port 8080 laufen. Er verarbeitet ausschliesslich nackte Zahlenwerte (Integers). Es werden keine Text- oder Rezeptdaten an die Hardware uebermittelt, um die Firmware so minimalistisch und stabil wie moeglich zu halten.

POST /fahre

Beschreibung: Wird aufgerufen, wenn der BarBot eine Linearachse/Schlitten bewegen und ein Servo-Ventil oeffnen soll.

Erwarteter Request-Body (JSON):

{
  "position": 4,   
  "menge": 50      
}

Erwartete Antwort (Response):

Der ESP muss die mechanische Aktion komplett abschliessen, bevor er antwortet. Nach Erfolg sendet er:

  • HTTP 200 OK mit Body: {"status": "done"}
POST /pumpe

Beschreibung: Wird aufgerufen, um eine stationaere Fluessigkeitspumpe per Relais fuer eine bestimmte Menge anzusteuern.

Erwarteter Request-Body (JSON):

{
  "pumpen_nr": 2,  
  "menge": 120     
}

Erwartete Antwort (Response):

Der ESP laesst die Pumpe laufen. Erst wenn der Pumpvorgang beendet ist, antwortet er:

  • HTTP 200 OK mit Body: {"status": "done"}

🌐 Interne Web-App API (Flask-Endpunkte)

Diese Endpunkte werden vom Frontend aufgerufen, um den Zustand des Mixvorgangs zu verwalten.

POST /api/order

Beschreibung: Startet einen neuen Mixvorgang fuer einen spezifischen Cocktail.

Request-Body (JSON):

{ "cocktail_id": 1 }

Antwort bei Erfolg (HTTP 200):

{ "status": "started", "total_steps": 3 }
GET /api/next_step

Beschreibung: Holt den naechsten Schritt aus der Warteschlange, triggert den ESP via POST und antwortet dem Frontend.

Moegliche Antworten (HTTP 200):

{
  "status": "success",
  "zutat_added": "Orangensaft",
  "progress": 1,
  "total": 3
}
