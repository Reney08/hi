BarBot Entwickler-Referenz¶

Dieses Kapitel ist in drei Bereiche unterteilt, je nachdem, ob du am Kern-System, der Hardware-Kommunikation oder der Web-Oberfläche arbeitest.

1. Interne Core-Logik & Datenbank¶

Hier werden alle internen Python-Funktionen dokumentiert, die für Datenbankabfragen, Berechnungen und die Verwaltung zuständig sind.

app.run_admin_app()[Quellcode]¶

Startet den Server für das Admin-Backend auf Port 1337.

app.run_user_app()[Quellcode]¶

Startet den Server für die Gäste-UI auf Port 5000.

2. ESP Hardware-Schnittstelle (API zum ESP)¶

Hier sind alle internen Funktionen gebündelt, die für die Kommunikation mit dem Mikrocontroller verantwortlich sind. Das Frontend ruft diese Funktionen indirekt über die REST-API auf.

esp_client.fahre_zu_position(schienen_pos: int, menge: int) → bool[Quellcode]¶

Sendet den Fahr-Befehl an den ESP (ohne Pumpenaktivierung). Kann genutzt werden, um eine Pause an einer bestimmten Position einzulegen.

Beispiel-Payload (JSON), das an /fahre gesendet wird:

{
    "schienen_pos": 5,
    "menge": 2000
}
Parameter:

  • schienen_pos (int) – Die Ziel-Position auf der Schiene.

  • menge (int) – Menge/Dauer, die an der Position gewartet wird.

Rückgabe:

True bei Erfolg, sonst False.

Rückgabetyp:

bool

esp_client.sende_heimfahrt() → bool[Quellcode]¶

Beordert den Schlitten zurück an die Startposition (Position 0). Ruft intern fahre_zu_position(0, 0) auf.

Erzeugtes Payload (JSON):

{
    "schienen_pos": 0,
    "menge": 0
}
Rückgabe:

True bei Erfolg, sonst False.

Rückgabetyp:

bool

esp_client.sende_led_modus(modus: str, position: int = None, r: int = 0, g: int = 0, b: int = 0, leds_top: list = None, leds_bottom: list = None) → bool[Quellcode]¶

Sendet einen LED-Steuerungsbefehl an den ESP zur visuellen Statusanzeige. Unterstützt verschiedene Modi wie einheitliche Farben, vordefinierte Effekte oder exakte LED-Mappings.

Beispiel-Payload (JSON) für eine positionsabhängige LED-Steuerung:

{
    "modus": "position",
    "leds_top": [90, 91, 92, 93, 94],
    "leds_bottom": [59, 58, 57, 56, 55]
}
Parameter:

  • modus (str) – Der gewünschte LED-Modus (z. B. ‚rainbow‘, ‚einheitlich‘, ‚position‘, ‚aus‘, ‚fertig‘).

  • position (int) – Optional; ID oder Index der physischen Flaschenposition.

  • r (int) – Rot-Wert (0-255) für den einheitlichen Farbmodus (Default: 0).

  • g (int) – Grün-Wert (0-255) für den einheitlichen Farbmodus (Default: 0).

  • b (int) – Blau-Wert (0-255) für den einheitlichen Farbmodus (Default: 0).

  • leds_top (list[int]) – Liste der exakten LED-Indizes der oberen Reihe, die leuchten sollen.

  • leds_bottom (list[int]) – Liste der exakten LED-Indizes der unteren Reihe, die leuchten sollen.

Rückgabe:

True bei erfolgreicher HTTP-Übermittlung, sonst False.

Rückgabetyp:

bool

esp_client.starte_reinigung(pumpen: List[int], dauer_sekunden: int = 20) → bool[Quellcode]¶

Sendet den Befehl zur Reinigung mehrerer Pumpen gleichzeitig. Gibt dem ESP automatisch ein längeres Timeout, da das Spülen Zeit kostet.

Beispiel-Payload (JSON), das an /reinigen gesendet wird:

{
    "pumpen": [1, 3, 4],
    "dauer_sekunden": 30
}
Parameter:

  • pumpen (list[int]) – Liste der zu reinigenden Pumpen-IDs.

  • dauer_sekunden (int) – Wie lange gespült werden soll (Default: 20s).

Rückgabe:

True bei Erfolg, sonst False.

Rückgabetyp:

bool

esp_client.steuere_pumpe(schienen_pos: int, pumpen_nr: int, menge: int) → bool[Quellcode]¶

Sendet den Befehl an den ESP, zu einer Position zu fahren und eine Pumpe zu aktivieren.

Beispiel-Payload (JSON), das an /pumpe gesendet wird:

{
    "schienen_pos": 3,
    "pumpen_nr": 2,
    "menge": 5000
}
Parameter:

  • schienen_pos (int) – Die Ziel-Position auf der Schiene.

  • pumpen_nr (int) – Die ID des Relais/der Pumpe.

  • menge (int) – Die zu pumpende Menge (z. B. in ml oder Millisekunden).

Rückgabe:

True wenn der ESP mit Status 200 antwortet, sonst False.

Rückgabetyp:

bool

3. Web REST-API (Für Frontend & Erweiterungen)¶

Hier findest du alle verfügbaren URLs, erwartete JSON-Payloads und Statuscodes für die Web-Interfaces.

Admin-Schnittstelle (Port 1337)¶

GET /admin/led¶

Rendert die Steuerungsseite für die LED-Beleuchtung.

POST /api/admin/led¶

Schaltet den globalen LED-Modus um.

Gäste-Schnittstelle (Port 5000)¶

GET /¶

Rendert die Haupt-Rahmenseite für Gäste (SPA Shell).

GET /home_content¶

Rendert die dynamischen Willkommens-Inhalte für die Startseite.

GET /cocktails¶

Rendert die Cocktail-Auswahlseite anhand der Datenbank.

GET /finished¶

Rendert die Erfolgsseite nach einem beendeten Mixvorgang.
